I’m Michał Łęcicki (aka Maikhel) - software engineer at Visuality.
I believe self-improvement, responsibility and honest code are necessary to get things done.

My path

My programming journey started in 2012 with free "Introduction to Java 101" Coursera course. Excited by power of creating a program that can rotate an image, I got to more serious learning at university. I finished with Bachelor’s degree in IT in 2016, University of Cardinal Stefan Wyszyński in Warsaw.

In 2015, I started working in Weblify as a junior Ruby on Rails developer and in 2018 I moved to Visuality as a software engineer. Surrounded by experienced engineers, I learned and still improve best development practices, applying clean architecture and using best design patterns.

As a backend engineer I’m usually responsible for preparing Ruby on Rails API applications (currently in JSON:API standard), building software prototypes or creating third party integrations (eg. with Pipedrive). Doing devops stuff on Heroku or AWS also happens to me. Few technologies I met so far include Elasticsearch, Terraform, Beskar, Serverless, R, Golang.

In addition to blog posts, I made a Lightning talk about Ruby on Rails & R.
Recently I'm taking a closer look at Golang and home automations (Hue API, Alexa).

Some facts I learned

  • You can use VIM for writing in every programming language.
  • Doing production deployment on Friday is not the best idea.
  • Wrong communication in the team is responsible for at least 50% of bugs.
  • Awful documentation is still better than no documentation.
  • You have enough time. What you need is motivation.
