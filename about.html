I’m Michał Łęcicki (aka Maikhel) - software engineer at Visuality.
I believe self-improvement, responsibility and clean solutions are necessary to get things done: both in software and real life.

My path

My programming journey started in 2012 with free "Introduction to Java 101" Coursera course. Excited by creating a program that can rotate an image, I got to more serious learning at university. I finished with Bachelor’s degree in IT in 2016, University of Cardinal Stefan Wyszyński in Warsaw.

In 2015, I started working in Weblify as a junior Ruby on Rails developer and in 2018 I moved to Visuality as a software engineer. Surrounded by experienced engineers, I learned and still improve best development practices, applying clean architecture and choosing right design patterns.

As a backend engineer, I’m usually responsible for preparing Ruby on Rails API applications (currently in JSON:API standard), building software prototypes or creating third party integrations (eg. with Pipedrive). Doing devops stuff on Heroku or AWS also happens to me. Few technologies I met so far include Elasticsearch, Terraform, Beskar, Serverless, R, Golang.

Recently, I'm taking a closer look at Domain Driven Design and Event Storming.

In addition to blog posts, from time to time I make lightning talks:

  • Ruby on Rails & R
  • How to use AWS SES with Lambda in Ruby on Rails Application

Or other presentations, such as presentation about Event Storming on Miłośnicy Ruby Warsaw Meetup.
